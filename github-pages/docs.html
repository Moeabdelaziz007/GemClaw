Architecture Overview

GemClawOS represents a paradigm shift in AI integration, moving from passive tools to sovereign, proactive entities. The system relies on a decentralized Next.js 15 frontend architecture combined with a high-performance Go backend and Firebase Firestore for persistent semantic memory.

The Neural Bridge

At the center of GemClawOS is the Neural Spine, facilitating ultra-low latency, full-duplex communication between the user's voice inputs and the Gemini Live API.

lib/store/useGemclawStore.ts

Live API Integration

The system connects to Gemini Multimodal endpoints via a secure WebSocket orchestration layer. All connections are routed through Edge functions to prevent client-side key exposure.

Authentication Flow

  1. Client requests brief lifecycle token via /api/gemini-token
  2. Firebase Auth verifies the session integrity
  3. Client establishes WebRTC/WebSocket using the ephemeral token
🛡️

Security Protocols

AetherOS emphasizes absolute security in the browser environment.

  • Zero Client Secrets: Never expose GEMINI_API_KEY to the window object.
  • CSP Enforced: Content Security Policy blocks arbitrary external script execution.
  • Manifest Validation: Agent capabilities (skills and tools) are cryptographically signed and checked at runtime before execution.
